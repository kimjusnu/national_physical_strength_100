# Claude Code 2차 개발 프롬프트 — 체력나이 웹앱

1차(랜딩→측정→결과, 목업)가 완료된 상태에서 이어서 붙여넣으세요.

---

1차 구현(입력 위저드 + 체력나이 산출 + 결과 화면, 목업)이 완료됐다. 이제 2차 기능을 이어서 구현한다. 기존 구조(`lib/kspoApi.ts` 추상화, `USE_MOCK` 플래그, 서버사이드 프록시)를 유지하고, 각 기능마다 목업으로 먼저 완성한 뒤 실 API 연동으로 넘어간다.

## 0. 체력나이 하한/상한 조정 (먼저)
- 현재 최상위권 기록이 15세까지 내려가 데모에서 어색하다. 클램핑 범위를 **20~70세**로 좁혀라.
- 산출된 체력나이가 하한(20세) 또는 상한(70세)에 도달한 경우, 숫자 대신/함께 **"상위 1% 체력" / "체력 개선 필요" 뱃지**를 표시해 극단값을 자연스럽게 처리한다.
- 관련 상수는 `lib/fitnessAge.ts` 상단에 모아 조정 가능하게 유지.

## 1. 약점 항목 기반 운동처방 영상 (동영상 API 15108846)
- 결과의 항목별 백분위에서 **하위 1~2개 약점 항목**을 뽑아, 그 항목을 개선하는 운동처방 영상 3~5개를 추천한다.
- `lib/kspoApi.ts`에 `getPrescriptionVideos(weakItems, ageGroup, gender)` 추가. 목업(`data/mockVideos.ts`)은 실제 동영상 API 응답 필드(영상 제목, 카테고리, 썸네일/재생 URL, 대상 등)를 반영.
- `components/PrescriptionVideos.tsx`: 카드형 리스트, 썸네일+제목+"왜 이 운동인지" 한 줄 설명(약점 항목과 연결). 클릭 시 새 탭 재생.
- 항목→운동 카테고리 매핑 규칙은 `lib/videoMatch.ts`로 분리.

## 2. 가까운 체력인증센터 안내 (측정건수/센터 API 15114286)
- 사용자가 지역(시도/시군구)을 선택하면 해당 지역 인증센터 목록을 보여준다. (위치기반은 선택: geolocation 있으면 가까운 순, 없으면 지역 선택.)
- `lib/kspoApi.ts`에 `getCenters(region)` 추가. 목업(`data/mockCenters.ts`)은 API 필드(센터명, 주소, 지역, 측정건수 등) 반영.
- `components/NearbyCenters.tsx`: 센터명/주소/측정건수 카드. 주소 클릭 시 지도앱/검색 링크로 연결(외부 링크).
- "실제 측정으로 정확히 인증받기" CTA로 온라인→오프라인 연계 메시지를 넣어 발전가능성 스토리를 강화.

## 3. 결과 카드 공유 이미지 (바이럴)
- 결과를 세로형 카드 이미지로 만들어 저장/공유. `lib/share.ts`에 canvas 또는 html-to-image 기반 `generateResultCard(result)` 구현.
- 카드 구성: 체력나이 큰 숫자 + 실제나이 대비 문구 + 상위 뱃지 + 항목 요약 미니차트 + 서비스명/URL 워터마크.
- `components/ShareCard.tsx`: "이미지 저장", "링크 복사"(현재 결과 URL), Web Share API 지원 시 공유 버튼.
- 접근성: 큰 글씨 모드에서도 카드 레이아웃이 깨지지 않게.

## 4. 실 API 연동 (측정결과 15108938)
- `.env.local`의 `USE_MOCK`을 `false`로 두면 실제 공공데이터포털 API를 호출하도록 전환. 서버사이드 Route Handler에서 `DATA_GO_KR_SERVICE_KEY`로 호출하고, XML/JSON 응답 파싱.
- 측정결과 API의 원자료를 **성별×연령대×항목별 백분위 기준테이블로 집계**하는 스크립트(`scripts/buildNormTables.ts`)를 작성해, 그 결과로 `data/normTables.ts`를 교체할 수 있게 한다. (API가 대량 원자료를 안 주면, 받아지는 통계 범위 내에서 최대한 집계하고 한계는 주석으로 남긴다.)
- 실 API 실패/지연 시 목업으로 graceful fallback. 타임아웃·에러 핸들링·간단한 캐싱(revalidate) 적용.
- 세 API 모두 승인 완료 상태이며 키는 이미 `.env.local`에 있음.

## 진행 방식
0 → 1 → 2 → 3 까지 목업 기반으로 완성해 결과 화면에 모두 통합한 뒤, 마지막에 4(실 API)를 연동한다. 각 단계마다 기존 테스트가 깨지지 않게 하고, 새 로직(videoMatch, centers, share)에 단위 테스트를 추가해줘. 먼저 0~1을 구현하고 결과를 보여줘.
