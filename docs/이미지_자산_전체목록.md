# 이미지 자산 전체 목록 · 생성 프롬프트

체력나이 서비스에 필요한 이미지 **총 26장**. 우선순위 순으로 정리했습니다.

생성 후 아래 표의 경로에 그대로 넣으면 됩니다. 파일명·크기를 지키는 게 중요합니다 — Next.js가 파일명으로 자동 인식하는 항목이 있습니다.

## 공통 규칙 (전 항목 공통)

```
Flat vector illustration, minimal geometric style. Clean white background
(#FFFFFF). Figures and objects in muted teal-slate (#34464B). One accent color
only: mint green (#04C584). No gradients, no drop shadows, no textures, no
photographic elements. NO TEXT, NO LETTERS, NO NUMBERS anywhere in the image.
Generous negative space. Calm and instructional tone, not energetic or
sporty-aggressive. Consistent line weight.
```

**텍스트 절대 금지** — 이미지 생성 AI는 한글을 깨진 글자로 그립니다. 모든 문구는 코드에서 얹습니다.

**색 고정** — 서비스가 흰 캔버스 + 틸-슬레이트 + 민트그린 단일 강조색 체계입니다. 다른 색이 섞이면 화면에서 이미지만 떠 보입니다.

# 1순위 — 브랜드 아이콘 (3장)

브라우저 탭·북마크·홈 화면에 뜨는 마크입니다. **16px에서도 알아볼 수 있어야** 하므로 디테일을 넣지 마세요.

## 디자인 컨셉

민트그린(`#04C584`) 배경에 흰색 인물 실루엣 하나. 이유: 16px로 줄여도 형태가 남고, 브랜드 색이 탭에서 바로 인지됩니다.

### 1-1. `app/icon.png` — 512×512

```
[공통 규칙 적용하되 배경만 예외] Solid mint green (#04C584) square background,
full bleed. Centered white silhouette of a single standing human figure with
arms slightly raised outward in a confident, balanced posture. The figure is
bold and simple, readable at very small sizes. Figure occupies about 60% of the
canvas height, centered with even margins. Pure white (#FFFFFF) figure, no
outline, no detail lines, no facial features. Flat, no gradient, no shadow.
Square 1:1 aspect ratio.
```

### 1-2. `app/apple-icon.png` — 180×180

1-1과 **완전히 같은 디자인**으로 180×180 생성 (또는 1-1을 리사이즈). iOS가 모서리를 자동으로 둥글게 깎으므로 가장자리 10%에는 중요한 요소를 두지 마세요.

### 1-3. `app/favicon.ico` — 32×32

1-1을 32×32로 축소해 `.ico`로 변환. 이 크기에서 형태가 뭉개지면 인물 실루엣을 더 단순하게 다시 뽑아야 합니다.

> **변환 도구**: [realfavicongenerator.net](https://realfavicongenerator.net) 에 512px 원본 하나만 올리면 ico 포함 전 사이즈를 만들어 줍니다.

# 2순위 — 소셜 공유 OG 이미지 (1장)

카카오톡·문자·SNS에 링크를 붙였을 때 뜨는 미리보기 카드입니다. 결과 공유가 이 서비스의 바이럴 경로라 **비어 있으면 손해가 큽니다.**

### 2-1. `app/opengraph-image.png` — 1200×630

```
[공통 규칙] Wide horizontal composition, 1200x630. Left two-thirds is empty
white space reserved for text overlay. Right third shows a minimal flat
illustration: a simple human figure standing beside an abstract vertical bar
chart of three bars in ascending height, the tallest bar in mint green
(#04C584), the other two in pale teal (#A7C7CF). Thin mint green horizontal
baseline runs under the figure and bars. Absolutely no text, no numbers, no
letters. Lots of clean white space.
```

**중요**: 왼쪽 2/3를 비워두라고 한 이유는, 거기에 "체력나이" 문구를 **제가 코드로 얹을 것**이기 때문입니다. 이미지에 한글을 넣으면 깨집니다.

> **더 나은 대안이 있습니다.** 결과 페이지의 OG 이미지는 사용자마다 다른 숫자("체력나이 31세")가 찍혀야 공유 효과가 큽니다. 이건 AI 이미지로는 불가능하고 코드로 동적 생성해야 합니다. 제가 구현할 수 있으니, 위 2-1은 **랜딩 페이지용 기본 이미지**로만 쓰고 결과 페이지는 저에게 맡기시면 됩니다.

# 만들지 않을 것 (의도적으로 제외)

요청하시면 만들 수 있지만, **넣으면 오히려 손해**라고 판단한 항목입니다.

| 항목 | 제외 이유 |
|---|---|
| 랜딩 히어로 일러스트 | 디자인 시스템이 "데이터가 곧 비주얼"입니다. 큰 장식 그림이 들어가면 공공데이터 서비스의 신뢰감이 흐려지고, 헬스 앱 광고처럼 보입니다. |
| 빈 상태(empty state) 일러스트 | DESIGN.md §14에서 빈 상태는 문장 한 줄 + CTA로 정의했습니다. 그림을 넣으면 오류 상황이 가벼워 보입니다. |
| 결과 공유 카드 배경 이미지 | 그 카드의 주인공은 사용자의 숫자입니다. 배경 그림이 들어가면 숫자의 힘이 죽습니다. 현재 canvas로 데이터를 그리는 방식이 맞습니다. |
| 체력요인 카테고리 아이콘 | lucide-react 아이콘을 이미 쓰고 있고, 라이브러리 아이콘이 손으로 만든 것보다 일관성이 높습니다. |
| 팀·회사 로고 | 개인/팀 출품작이라 불필요합니다. 워드마크("체력나이")는 텍스트로 렌더 중입니다. |

# 전달 방법

폴더에 넣어주시거나, 파일만 주시면 제가 배치하겠습니다.

```
app/
  icon.png              512×512
  apple-icon.png        180×180
  favicon.ico            32×32
  opengraph-image.png  1200×630
public/
  guides/
    grip.webp  situp.webp  jump.webp  bodyfat.webp  shuttle.webp   각 800×600
  thumbnails/
    01.webp ~ 15.webp                                              각 480×320
```

파일을 받으면 제가 할 일: 안내 모달에 그림 삽입, OG 메타데이터 설정, 썸네일 경로 연결, 결과 페이지 동적 OG 구현. 코드 쪽 자리는 이미 만들어 두었으니 파일만 오면 바로 붙습니다.

**포맷**: webp 우선. png로 주셔도 되고, 그 경우 확장자만 맞춰 알려주세요.
