## ① 기준분포 집계 — 측정결과 정보(15108938)

체력나이 산출의 근간. 공단 측정결과를 **성별×연령대×항목별 평균·표준편차**로 집계해 기준선을 만들고, 사용자 값을 그 분포 위에 올려 백분위와 체력나이를 역산합니다.

| # | 파일 | 위치 | 역할 |
|---|---|---|---|
| 1 | `data/normTables.ts` | L4 (주석), L53 `NORM_TABLES` | 성별×연령대(7구간)×항목(5종) 기준분포 테이블. 체력나이 산출의 기준선 |
| 2 | `data/mockMeasurements.ts` | L2 | 측정결과 API 응답 스키마를 반영한 원시 레코드 시드 |
| 12 | `lib/normAggregate.ts` | L6 (주석), L50 `aggregateNormTables()` | **원시 측정 레코드 → 기준테이블 집계 로직** |
| 3 | `lib/fitnessAge.ts` | L15 (주석) | 기준분포를 사용한 체력나이 산출 |
| 4 | `app/result/page.tsx` | L41 | 결과 화면에서의 산출 호출부 |

**핵심 함수**

```
lib/normAggregate.ts
  L50  aggregateNormTables(records)   원시 레코드를 성별×연령대×항목별로 집계
  L30  mean(values)                    구간별 평균
  L36  stdDev(values)                  표본표준편차 (n-1)
  L17  MIN_SAMPLES_PER_CELL = 30       표본 부족 셀은 기존 기준값 유지 + 경고

lib/fitnessAge.ts
  L49  percentileFor(norm, age, value) 사용자 값 → 같은 성별·연령대 분포 내 백분위
  L60  itemFitnessAge(norm, value)     "이 성적이 평균이 되는 나이" 역산 (구간 선형보간)
  L84  computeFitnessAge(user)         항목별 체력나이의 가중평균 → 종합 체력나이
  L24  FITNESS_AGE_MIN = 20 / MAX = 70 산출 범위 클램핑

scripts/buildNormTables.ts             실 API 원자료를 수집·집계해 기준테이블 생성
```

**산출 3단계**
1. **백분위 환산** — 사용자 값을 성별·연령대 분포(정규분포 가정)의 백분위로 변환
2. **항목별 체력나이 역산** — 연령별 평균 곡선에서 그 성적이 평균이 되는 나이를 선형보간
3. **종합 체력나이** — 항목별 체력나이의 가중평균 (`ITEM_WEIGHTS`, `data/normTables.ts` L80)

**데이터 신뢰성 처리**: 셀당 표본이 30건 미만이면 집계값 대신 공개 기준값을 유지하고 경고를 남깁니다. 부족한 표본이 기준선을 왜곡하는 것을 막기 위함입니다(`lib/normAggregate.ts` L17, L112).

## ③ 체력인증센터 조회 — 측정건수 정보(15114286)

| — | `app/api/centers/route.ts` | L8 `GET()` | 서버사이드 프록시 — 인증키를 클라이언트에 노출하지 않음 |

## 공통 — 데이터 계층 추상화

`lib/kspoApi.ts` 한 곳에서 세 API를 모두 다룹니다.

| 기능 | 위치 |
|---|---|
| 목업/실API 전환 (`USE_MOCK`) | L36 `isMockEnabled()` |
| 인증키 관리 | L40 `serviceKey()` — 미설정 시 즉시 예외 |
| 공통 조회 + 타임아웃 8초 | L52 `fetchKspoItems()` |
| 실패 시 시드 폴백 | 세 함수 모두 `catch`에서 처리 |

**가용성 설계**: 실 API 실패·지연 시 오류를 노출하지 않고 시드 데이터로 자동 대체합니다. 심사위원이 접속했을 때 화면이 비는 상황을 막기 위한 조치입니다.
