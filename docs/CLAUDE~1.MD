## 프로젝트 개요

국민체육진흥공단(KSPO) 공공데이터 활용 경진대회 출품용 **반응형 웹앱**을 만든다.
서비스명(가칭): **"체력나이"(Fitness Age)** — 사용자의 체력을 측정/입력받아, 국민체력100 공공데이터 분포와 비교해 "당신의 체력 나이는 OO세"를 산출하고, 맞춤 운동처방 영상과 가까운 체력인증센터를 안내한다.

이 서비스는 심사(서류)에서 다음 배점을 최적화하도록 설계한다: 독창성 25%, 실용성 20%, 적절성(데이터활용) 20%, 발전가능성 20%, 기술성 15%. 따라서 **국민체력100 공공데이터를 서비스의 핵심 로직으로 명확히 사용**하고, **결과 공유 기능으로 바이럴/실용성**을 살리고, **완성도 높은 단일 플로우**로 구현한다.

## 기술 스택
- Next.js (App Router) + TypeScript + Tailwind CSS
- 상태관리는 기본 React 훅, 차트는 recharts
- 배포는 Vercel 기준 (환경변수로 API 키 관리)
- 모바일 우선 반응형. 접근성(큰 글씨 토글) 고려 — 전 연령 대상

## 핵심 사용자 플로우 (한 화면씩 이어지는 위저드)
1. **랜딩**: 서비스 소개 + "내 체력나이 측정하기" CTA. 데이터 출처(KSPO 국민체력100)를 명시.
2. **기본정보 입력**: 성별, 나이(실제), 키, 몸무게.
3. **체력 항목 입력**: 국민체력100 측정 항목을 사용자가 직접 입력하거나, 화면 안내에 따라 간이 자가측정 후 값 입력.
   - 항목: 악력(kg), 윗몸말아올리기(회/1분), 반복옆뛰기 또는 제자리멀리뛰기, 체지방률(%) 또는 허리둘레(cm), (선택) 왕복오래달리기 단계.
   - 각 항목에 "측정 방법" 안내 모달(그림/텍스트)을 둔다.
4. **결과 화면**: 
   - 큰 숫자로 **체력나이** 표시 (실제나이 대비 +/- 강조, 색상 피드백).
   - 항목별 백분위 레이더/바 차트 (recharts).
   - 강점/약점 요약 문구.
   - **맞춤 운동처방 영상 3~5개** (국민체력100 동영상 API 결과, 약점 항목 기준 추천).
   - **가까운 체력인증센터** 안내(측정건수/센터 정보 데이터 활용, 지역 선택 또는 위치 기반).
   - **결과 카드 이미지 공유/저장** 버튼 (SNS 바이럴용).

## 체력나이 산출 로직 (핵심 알고리즘)
- 국민체력100 측정결과 데이터로 **성별×연령대×항목별 기준분포(백분위 테이블)**를 구성한다.
- 사용자의 각 항목 값을 자신의 성별 분포에서 백분위로 환산.
- 각 항목에서 "사용자의 성적이 평균이 되는 연령대"를 역산 → 항목별 체력나이.
- 항목별 체력나이의 가중평균으로 **종합 체력나이** 산출 (가중치는 상수로 분리해 조정 가능하게).
- 로직은 `lib/fitnessAge.ts`에 순수함수로 분리하고, 기준 테이블은 `data/normTables.ts`(JSON)로 분리한다.

## 데이터 소스 (공공데이터포털 오픈API)
개발 초기엔 API 키 발급 전이므로 **실제 API 스키마를 반영한 시드/목업 데이터**로 먼저 완성하고, 이후 실 API로 교체할 수 있게 데이터 계층을 추상화한다(`lib/kspoApi.ts` 인터페이스).

- 국민체력100 체력인증센터 측정결과 정보: data.go.kr/data/**15108938**/openapi.do (항목: 신장, 체중, 체지방률, 허리둘레, 혈압, 악력, 윗몸말아올리기, 반복점프 등 + 연령/성별)
- 국민체력100 체력인증센터 측정건수 정보: **15114286** (지역별 센터/측정건수)
- 국민체력100 동영상 정보: **15108846** (운동처방/생애주기별 표준운동 영상 ~1천편)
- (선택) 전국체육시설 안전점검: 15107773

`.env.local`에 `DATA_GO_KR_SERVICE_KEY` 자리만 만들어두고, 목업↔실API 전환 플래그(`USE_MOCK`)를 둔다.

## 구현 우선순위 (핵심기능 집중 — 4주 MVP)
1. 입력 위저드 + 체력나이 산출 로직 + 결과 화면(차트 포함) — **끝까지 동작하는 한 플로우 완성 최우선**
2. 운동처방 영상 추천 (약점 항목 매핑)
3. 가까운 인증센터 안내
4. 결과 카드 공유 이미지
5. 실 API 연동 (키 발급 후)

## 완성 기준
- 데스크톱/모바일 반응형으로 랜딩→측정→결과 전 과정이 끊김 없이 동작.
- 목업만으로도 심사위원이 URL 접속해 결과를 볼 수 있어야 함(심사 필수 요건: 접속 가능한 서비스 URL).
- 코드에 국민체력100 데이터 활용 지점을 주석으로 명확히 표시(보고서 "데이터 활용 부분" 작성용).
- README에 데이터 출처, 실행법, 배포법 기재.

먼저 프로젝트 스캐폴딩 + 입력 위저드 + 체력나이 산출 로직 + 결과 화면까지 목업 데이터로 동작하도록 구현해줘. 완료되면 다음 단계(영상 추천)로 넘어간다.
